## 一、项目背景

构建一个**结构化 Markdown 转思维导图**的在线工具网站，让用户通过输入符合特定规范的 Markdown 文本，实时生成可交互的思维导图。

---

## 二、核心需求确认

### 2.1 功能需求

| 需求类别 | 具体需求 | 优先级 | 阶段 |
|---------|---------|--------|------|
| **输入** | Markdown 文本编辑器（支持实时编辑） | P0 | v1.0 |
| **解析** | 将 Markdown 解析为树形思维导图结构 | P0 | v1.0 |
| **渲染** | 可视化展示思维导图 | P0 | v1.0 |
| **交互** | 思维导图支持展开/折叠 | P1 | v1.0 |
| **交互** | 思维导图支持拖拽 | P1 | v1.0 |
| **交互** | 思维导图支持缩放 | P1 | v1.0 |
| **导出** | 导出为 PNG 格式 | P1 | v1.0 |
| **导出** | 导出为 SVG 格式 | P1 | v2.0 |
| **导出** | 导出为 JSON 格式 | P1 | v1.0 |
| **主题** | 预设主题切换（3-5种） | P2 | v1.0 |
| **主题** | 自定义主题配置 | P2 | v2.0 |
| **示例** | 提供 Markdown 语法示例和模板 | P2 | v1.0 |

### 2.2 非功能需求

- **性能**：支持 500+ 节点的流畅渲染
- **响应式**：支持桌面端为主，移动端适配
- **无依赖**：可离线使用
- **开源**：代码可维护、可扩展

## 三、技术方案

### 3.1 核心技术选型（B+C 结合）

| 层级 | 选型 | 说明 | 来源 |
|------|------|------|------|
| **框架** | Vue 3 + Composition API | 当前项目已有 | B |
| **构建** | Vite | 当前项目已有 | B |
| **Markdown 编辑器** | CodeMirror 6 | 功能丰富，可高度定制 | C |
| **Markdown 解析** | marked + 自定义插件 | 成熟 + 可扩展 | B |

| **样式** | CSS 变量 + SCSS | 主题系统支持 | B/C |
| **导出** | Canvas API / svg-to-png | PNG 导出 | B |
| **导出** | 原生 SVG 序列化 | SVG 导出 | C |

## 五、Markdown 语法规范

### 5.1 基础格式

```markdown
# 中心主题
## 一级分支
### 二级节点
### 二级节点
## 一级分支
### 二级节点
```

### 5.2 带图标的标题

```markdown
# 🌟 主标题
## 📚 分类一
### ✨ 要点一
### 💡 要点二
## 🎯 分类二
```

### 5.3 列表格式（备选）

```markdown
# 中心主题
- 分支一
  - 子节点1
  - 子节点2
- 分支二
  - 子节点3
```

## 九、附录：技术选型理由

### CodeMirror 6 vs 其他编辑器

| 编辑器 | 优点 | 缺点 |
|--------|------|------|
| CodeMirror 6 | 高度可定制、现代、插件丰富 | 学习曲线 |
| Monaco | 功能强大（VS Code 同款） | 体积大 |
| 原生 textarea | 无依赖 | 无功能 |

**结论**：选择 CodeMirror 6，平衡功能与体积

### 自研 SVG 渲染 vs 第三方库

| 方案 | 优点 | 缺点 |
|------|------|------|
| Mind-elixir | 功能完整 | 定制受限 |
| markmap | 专为 Markdown 设计 | 交互弱 |
| **自研 SVG** | 完全可控、易定制 | 工作量大 |

**结论**：v1.0 采用自研 SVG 渲染，为 C 方案奠定基础
