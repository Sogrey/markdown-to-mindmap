## GitHub Actions 编排
- `.github/workflows/deploy.yml` 合并为单文件，含三个触发场景：
  1. push main → build + deploy GitHub Pages
  2. pull_request main → build only（验证性构建）
  3. push tag v*.*.* → build + 打包 zip + 使用 `softprops/action-gh-release@v2` 发布 Release
- Release 包命名：`markdown-to-mindmap-{version}.zip`，从 tag 名自动提取版本号（去掉 `v` 前缀）
- `generate_release_notes: true` 自动生成变更摘要（基于 commit 历史）
- 共享 `build` job：Pages 和 Release 共用同一构建结果，避免重复构建
